# 🎯 Goal State — Definition of Done  *(locked 2026-06-21)*

*The contract. The autoplay + self-improvement loops run autonomously toward **this** until
every box is ticked. Locked with Jae after a two-round goal interview. "Done" is measured, not
felt.*

---

## The end state, in one breath
Open **any** chat, say *"be my DM,"* and play — **talking to it conversationally to update or
add things as I go.** The system has a **flagship ~30-minute Rules-Light campaign** (heroic
fantasy, **multiple branching storylines, every class matters**, a rich story) that **proves
the engine**: put good campaign detail in, and the game runs itself well. Plus **one complete
royalty-free campaign** (a real open-licensed adventure, fully detailed — choices, likelihoods,
story connections) and **a starter in each remaining system**. The engine **plays itself**
(autoplay) and **improves every run** (self-correction); once it clears the bar a few times,
**Jae plays one session to bless it.**

## Scope (locked)
- **Systems / sources:** 🎲 Rules-Light · 📚 Authentic D&D 5e · 🧩 LitRPG · 🧾 Royalty-free modules — all four in scope (generated originals assumed throughout).
- **Default content rating: Mature (R)** — graphic violence and adult themes allowed within
  per-campaign lines; `dial it back` / `fade to black` / `pause` always honoured instantly.
- **Autonomy:** run the loop unattended; **surface only at a gate boundary, a blocker, or a
  genuine taste call** (e.g. choosing the royalty-free source, a mature-content line).
- **Validation:** autoplay must hit the bar across several runs, **then Jae plays one session to bless it.**

## The deliverable campaign roster
1. **Flagship** — *Rules-Light · heroic fantasy · ~30 min.* Multiple branching storylines on a
   real spine with the Living-Map; **all 6 classes get a distinct, mechanically-meaningful
   moment** (no filler class); rich NPCs + companions. The showcase that proves "good detail
   in → great game out."
2. **Royalty-free complete campaign** — a vetted open-licensed (OGL/CC) published adventure,
   fully statted and wired into the engine (choices, DCs, likelihoods, story connections),
   **QA'd by agents several times with the self-correction loop** until results are solid.
3. **Per-system starters** — a short ready adventure in each remaining system (Authentic 5e,
   LitRPG). *(Rules-Light already has Smoke Over Bramblewick + the Ember Key test.)*

## Acceptance criteria (all must pass — each checkable)
1. **Turnkey cold-load** — any fresh chat + Notion → menu in one reply, no uploads.
2. **Conversational live-authoring** — mid-session "add X / change Y" updates canon + `state.json` + dashboard.
3. **Save/load fidelity** — exact reconstruction in a new chat (no invented numbers).
4. **Rules completeness** — the DM never stalls on an undefined rule, per system in scope.
5. **Coherence** — no contradictions; the Living-Map stays consistent and append-only.
6. **Meaningful branching** — the flagship offers ≥3 genuinely different routes/endings; the map diverges.
7. **Every class matters** — in the flagship, each class has a distinct mechanical + narrative moment.
8. **Fairness / balance** — odds in band; lethality has real stakes (Mature) but fail-forward by default.
9. **Alive party** — distinct companions that advise + **vote** at forks; approval gates content; personas in the save.
10. **Onboarding** — a newcomer is playing in ~2 min; ready adventures are menu-selectable.
11. **Live dashboard loop** — `state.json` → push → render, end-to-end, ≤ ~1 min.
12. **Self-improvement loop** — every save → AAR → fold; rubric scores trend up.
13. **Autoplay convergence** — each campaign meets the decent-results bar below.
14. **Content depth & legal** — the roster exists; SRD attributed; LitRPG original homage; royalty-free properly licensed; nothing in any work repo.

## The "decent results" bar (the loop's stop condition)
A campaign **passes** when an autoplay batch scores **≥4/5 on all six rubric axes** (Runnable,
Coherent, Branching, Fair, Alive, Onboardable) **across ≥3 consecutive runs** with varied
player dispositions and **zero "invented a core rule" flags**. The **flagship** and the
**royalty-free** campaign must each pass; then Jae blesses the flagship with one human session.

## The gates (autonomous run order)
- **Gate 1 — Engine proven (micro).** ✅ **MET** — Ember Key autoplays, branches, self-corrects.
- **Gate 2 — Nuance + a runnable party.** Fold the Story spine / Freedom dial + the Rules-Light
  rule gaps + `persona` into canon; autoplay a **party** scene (Bramblewick) with companion
  **voting** to the bar.
- **Gate 3 — The flagship to confidence.** Build the 30-min Rules-Light heroic flagship (spine +
  branches + all-6-classes-matter); autoplay to the bar; **Jae plays it to bless.**
- **Gate 4 — Breadth + the royalty-free campaign.** Add the royalty-free complete campaign
  (vetted, statted, QA'd ×N) + a starter per remaining system; turnkey cold-load passes; all four systems exercised.

## Done-done
All 14 criteria pass; the flagship **and** the royalty-free campaign each clear the
decent-results bar; Jae has blessed the flagship with one human session. Then: open any chat,
say *"be my DM,"* talk to it freely — and it just works, well.

## How I run it (protocol)
Per gate: **build / fix → autoplay (≥3 varied runs) → Critic scores vs the rubric → fold
learnings (`./SELF-IMPROVEMENT.md` → `./LEARNINGS.md`) → repeat until the bar clears.** Surface
at each gate with a short report, on a blocker, or for a taste call. Commit to the repo (and
Notion where relevant); pushes are **merge-not-force** to stay safe alongside parallel sessions.
